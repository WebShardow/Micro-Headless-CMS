# Micro Headless CMS - คู่มือการอัปเดตและติดตั้งใหม่ (2026-03-29)

คู่มือนี้สรุปขั้นตอนที่จำเป็นหลังจากมีการนำโค้ดล่าสุดจาก `Micro-Team-Backend` มายัง CMS ชุดหลักนี้ โดยครอบคลุมทั้งการจัดการไฟล์และฐานข้อมูล

---

## 🚀 ขั้นตอนการติดตั้ง/อัปเดต

### 1. การเตรียมสภาพแวดล้อม
- ย้ายไฟล์โปรเจกต์มายังโฟลเดอร์นี้ **ยกเว้นโฟลเดอร์ `.git` และไฟล์ `.env`**
- ตรวจสอบไฟล์ `.env` ให้แน่ใจว่าเชื่อมต่อกับ Database เดิมที่ต้องการ

### 2. ติดตั้ง Dependencies
ใช้คำสั่งด้านล่างเพื่ออัปเดต Library ใหม่ๆ (รวมถึง Prisma Client):
```bash
npm install
```

### 3. อัปเกรดโครงสร้างฐานข้อมูล (Database Schema)
เนื่องจากมีการเพิ่มตารางใหม่ (เช่น `StaffRepo` สำหรับจัดเก็บพอร์ตโฟลิโอทีมงาน) และฟิลด์ใหม่ใน `SiteConfig` (ลิงก์โซเชียลและกฎหมาย) ให้รันคำสั่ง:
```bash
npx prisma db push
```
*หมายเหตุ: คำสั่งนี้จะ Sync โครงสร้างตารางให้ตรงกันโดยไม่ลบข้อมูลที่มีอยู่ (ยกเว้นกรณีมีการเปลี่ยนชื่อตารางชัดเจน Prisma จะแจ้งเตือน)*

### 4. เตรียมเนื้อหาเริ่มต้น (Legal & Seeding)
ระบบใหม่ต้องการเนื้อหาสำหรับหน้า Privacy Policy, Terms, และ Cookies เพื่อใช้ในส่วน Footer ของ Frontend:
```bash
npx tsx scripts/update-legal-content.ts
```

---

## 🛠 ฟีเจอร์ใหม่ที่บรรจุเพิ่มเข้ามา
- **Admin Configuration:** รองรับการจัดการลิงก์ GitHub, Twitter, LinkedIn, Privacy, Terms, Cookies และเพิ่มตัวเลือกสี Accent Staff อีก 4 สี
- **Dynamic Legal Pages:** ระบบรองรับการสร้างและแก้ไขหน้าเนื้อหาแบบ Dynamic ผ่านเมนู `Pages`
- **Staff Repository Manager:** ระบบจัดการพอร์ตโฟลิโอเชิงลึกสำหรับสมาชิกแต่ละคนในทีม
- **Improved API:** รองรับการ Fetch หน้าเพจผ่าน `slug` โดยตรง (แก้ไขข้อผิดพลาด 405/404)

## ✅ การตรวจสอบความสมบูรณ์
รันคำสั่ง Build เพื่อเช็คว่าไม่มีไฟล์ใดสูญหายหรือผิดพลาด:
```bash
npm run build
```
หาก Build สำเร็จ (Exit code: 0) แสดงว่าระบบพร้อมให้บริการ 100% ครับ
